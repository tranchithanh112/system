1.0.2.30 (2010-11-18A)

  • Improved filter error in case opening global variable editor with empty variable list.
  • Added new functions for managing date and time: SET_YEAR, SET_MONTH, SET_DAY, SET_HOUR, SET_MINUTE, SET_SECOND, SET_HUNDSEC, DATE_TIME_TO_DATE.
  • Improved type DATE handling in CPSim.

1.0.2.29 (2010-11-16B)

  • Updated compiler to 0.4.11.24 version:
    • Added (*$HIDDEN*) directive for hidding global variables in compilation report. It can be applied like RETAIN or CONSTANT modifiers.
  • Added columns for global variable editor: Initial value and comment.
  • Added buttons for moving (ordering) variables in global variable editor.
  • Added fast seeking to variable in global variable editor by name.
  • Added checking for empty directory before saving project. It can be disabled by setting CheckForEmptyDirWhenSaving configuration option to False value.

1.0.2.28 (2010-11-04B)

  • Firmware SMC has been updated to 1.38 version and CPCon to 1.1.38 version.

1.0.2.27 (2010-10-06A)

  • Updated compiler to 0.4.11.23 version:
    • Added new flag for compiler which enables different interpretation of global variables like in IEC 61131-3 standard. It influcences on VAR CONSTANT delared locally (now with this flag it should be declared following: VAR_GLOBAL CONSTANT, otherwise constants will not be visilbe everywhere as global).
    • Fixed situations when global constant value was hidded by (*$AUTO*) directive.
    • Added END_FOR checking before end of POU code.
  • Added option in configuration window to enable VAR_GLOBAL processing in accordance to IEC 61131-3 standard.
  • Added sorting in context menu for global variables tree (alphabetically ascending/descending), and restoring declaration (natural) order.
  • Fixed ABS and NE functions with FINT type.

1.0.2.26 (2010-09-08A)

  • Updated compiler to 0.4.11.22 version:
    • Fixed bug with removing function block body when CONST_INIT_INSTANCE directive was present.
    • Problem with misplaced nested types when source code is invalid has been localized. Temporary solution adds prefix "!Check code {typ}" for short name.
  • New function block for SM3 control published as COM_SM3. Previous COM_SM3 have changed name to COM_SM3_SIMPLE.
  • New context menu cut / copy / paste in STColorable editor.
  • Changed comments in IEC_61131 library and Basic_blocks library to English.

1.0.2.25 (2010-07-21B)

  • Updated compiler to 0.4.11.20 version:
    • Modified tasks structure for multitasking virtual machines,
    • Added index value checking in arrays when indexer declared as constant,
    • Changed missing semicolor warning to error.
  • Added separate context menu for global variables.
  • Added new keyoboard shortcut Ctrl + Del for removing all messages in bottom list.
  • Added two options for name searching - selecting variable in object tree and using context menu, or via menu 'Project | Tools | Localize identifier' and giving the identificator name. Localized references are presented in bottom message list. Double click on them opens editor and moves the cursor to the begin of the identifier. Choosing global variable and using context menu skips itself declaration in references list.
  • Updated compiler to 0.4.11.21 version:
    • New directive NO_ALIGN_VAR for disabling alingment of local variables in funtion block. Before it was implicated by existment of all following directives: CONST_INIT_INSTANCE, PLACE_INSTANCES_IN_DCP, EMPTY_BODY_CALL.

1.0.2.24 (2010-06-23A)

  • Updated compiler to 0.4.11.19 version:
    • Improved compiler hang when compiling expressions that contains boolean operators with invalid operands.
